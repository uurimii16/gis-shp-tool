## 2026-07-03

기존 3탭(좌표변환·병합·분할) 기반 앱에 아래 기능을 추가·검증했습니다. 모든 항목은 로컬 QGIS 내장 GDAL 3.7로 실데이터 end-to-end 검증 완료.

### 신규 기능
- **변환 전후 feature count / extent 비교** — `ogrinfo` 출력을 파싱해 변환 전후 피처 수와 범위를 표로 비교.
- **도형 유효화 `-makevalid` 옵션** — 변환·병합·분할 공통. self-intersection 등 불량 도형 자동 보정.
- **병합(dissolve) 속성 집계 옵션** — 기준 컬럼 외 속성을 SUM/AVG/MAX/MIN/COUNT로 요약(컬럼별 지정).
- **작업 로그 다운로드** — 각 탭 처리 로그를 텍스트로 저장.
- **GDAL 미설치 안내 강화** — 설치/연결 방법을 사이드바에 단계별로 안내.
- **GPKG 내부 복수 레이어 선택** — `ogrinfo`로 GPKG 내부 레이어를 나열해 `파일명:레이어명` 단위로 개별 인식·선택.
- **면적 컬럼(area_m2, ㎡) 추가** — 변환·병합 탭. `ST_Area` 기반, 투영 후 결과에 후처리로 계산(미터 좌표계에서 정확). 경위도(4326)는 경고.
- **4. 코드 결합 탭(신규)** — SHP 속성의 MNUM 등에서 `substr(컬럼, 시작, 길이)`(기본 21,6)로 코드를 뽑아 용도지역 코드표 CSV를 LEFT JOIN. 원본 DBF를 직접 편집하지 않고 결합된 새 SHP/GPKG 생성.
  - 코드표 CSV 인코딩 후보별 깨짐 점수 표시 + 수동 선택(UTF-8-SIG/CP949/EUC-KR/UTF-8).
  - 대상 SHP 인코딩 확인, MNUM 컬럼/시작·길이 지정, 추출 코드·매칭률 미리보기.
- **안전 변환 모드(변환 탭, 기본 ON)** — 불안정 SHP를 다른 좌표계로 재투영할 때 폴리곤이 한두 개 사라지는 문제 방지.
  - ① 원본 좌표계 확정(`-a_srs`) + `makevalid`로 네이티브 좌표에서 도형 복구 → ② 재투영(`-t_srs`) + `makevalid` 2단계.
  - 입력 → 복구후 → 출력 단계별 피처 수 리포트 + 손실 발생 시 빨간 경고.

### 신규 기능: "속성만 합치기(도형·개수 유지)" 모드 (레이어 병합 탭)
- dissolve와 별개로, **도형을 전혀 안 바꾸고** 그룹 기준 컬럼별 값을 각 피처에 새 컬럼으로 붙이는 모드. 함수 `aggregate_keep_geometry`(자기조인: 그룹 집계 서브쿼리와 기준 컬럼 1:1 LEFT JOIN → 개수·fid 안전).
- **면적 옵션(기본 ON)**: 원본에 면적 열이 없어도 도형에서 계산. 각 폴리곤 `area_m2`(ST_Area) + 그룹별 `area_sum`. **area_sum은 `ST_Area(ST_Union(geometry))`로 '실제 합친 면적'을 구해 폴리곤이 겹쳐도 겹친 부분을 한 번만 셈**(단순 SUM은 겹침 중복→과대평가). union 안전 위해 ①단계에서 CLI `-makevalid`로 도형 복구(유효 도형은 불변, 클라우드 gdal-bin 호환 위해 ST_MakeValid SQL 함수 대신 CLI 사용). 면적 정확도 위해 ①단계에서 재투영(target_epsg).
- 컬럼명은 ASCII(`area_m2`/`area_sum`)로 — SHP 10바이트 필드명 제한에서 한글이 잘려 값이 깨지는 것 방지(검증: 한글명 `area_m2_합계`는 SHP에서 0으로 깨짐 확인 → ASCII로 수정).
- 속성 집계(SUM/AVG/MAX/MIN/COUNT/GROUP_CONCAT)도 선택 가능(면적과 별개, 선택).
- 검증(로컬 GDAL 3.7): 겹치는 자연녹지 2개→2개 유지 area_sum=175(정확), 흩어진 자연녹지 3개→3개 유지 area_sum=300. SHP/GPKG 둘 다 정상.

### ✅ 해결: dissolve 결과 폴리곤 도형 깨짐 (아래 🔴 이슈 종결)
- **수정**: `dissolve_one_layer`를 "복구 먼저 → union"의 **2단계**로 재작성(합의된 방향 3안 채택).
  1. **① 도형 복구**: 원본을 `ogr2ogr -makevalid`로 UTF-8 GPKG(`-nln src`, `-lco GEOMETRY_NAME=geometry`)로 복구. 재투영 없이 네이티브 좌표에서. spatialite `ST_MakeValid` SQL 함수에 의존하지 않고 GDAL의 `-makevalid`(GEOS MakeValid)를 사용해 클라우드 호환성 확보.
  2. **② union**: 복구된 GPKG에서 `ST_Union(geometry) ... GROUP BY` 실행 + 최종 `-makevalid`(항상) + 선택적 `-t_srs` 재투영. 이후 `sqlite_sql_to_output` 경로로 인코딩 안전 저장.
- **복구는 항상 수행**(makevalid 체크박스와 무관). 이것이 근본 해결이므로 기본 OFF로 두면 안 됨.
- **주의점(구현 중 발견)**: GPKG 지오메트리 컬럼 기본명은 `geom`이라 union SQL의 `FROM src`에서 `geometry`를 못 찾음 → 복구 단계에 `-lco GEOMETRY_NAME=geometry`로 컬럼명을 고정해 해결.
- **검증(로컬 QGIS GDAL 3.7, end-to-end)**: 위상 오류(자기교차 bowtie + 겹침) 폴리곤 테스트셋으로 비교.
  - 수정 전: 불량 그룹이 `ST_IsValid=-1`(GEOS TopologyException), 면적 null → 깨짐 재현.
  - 수정 후: 모든 그룹 `ST_IsValid=1`. 인접 정상 사각형 2개는 **외곽선 그대로 20×10 단일 폴리곤으로 병합**(링 1개, 내부 공유 경계만 제거, 바깥 윤곽 보존 확인). 속성만 합쳐지고 도형 모양은 불변.
- **UI**: 병합 탭 caption에 "바깥 윤곽 유지·내부 공유 경계만 제거·복구 자동 수행" 안내 추가.
- **남은 확인(배포 시)**: 클라우드 gdal-bin에서도 `-makevalid`/`GEOMETRY_NAME` 동작 확인. 사용자 실제 데이터(전주 정비 SHP)로 최종 눈검수 권장.

## 🔴 미해결 / 다음 작업 (2026-07-03 종료 시점) — ✅ 위 2026-07-06에서 해결됨

### 이슈: dissolve 결과 폴리곤 선형(geometry)이 깨짐
- **증상**: "한 SHP 내 컬럼값 기준 병합(dissolve)" 실행 시 텍스트(속성)는 정상인데 **폴리곤 경계선이 이상하게 깨짐**(스파이크/잘못된 선).
- **원인 확정**: 결과를 **GPKG로 출력해도 동일하게 깨짐** → GPKG→SHP 변환 문제가 **아님**. **dissolve의 `ST_Union` 연산 자체**가 원인. GPKG→SHP는 정점 좌표를 그대로 복사하므로 도형 왜곡 없음(곡선만 SHP 미지원인데 dissolve 결과는 직선이라 무관).
- **추정 세부원인**: 원본 폴리곤의 위상 오류(자기교차/겹침/미세 슬리버/링 방향)를 **복구하지 않고** `ST_Union`하면 GEOS가 깨진 결과를 냄. 현재 `dissolve_one_layer`는 `-makevalid`를 union **다음**에 적용(순서 비효율). makevalid 체크박스 기본 OFF라 대개 복구 없이 union됨.

### 다음에 할 수정 (합의된 방향)
1. **dissolve를 "복구 먼저 → union" 순서로**: SQL을 `ST_Union(ST_MakeValid(geometry))` 형태로 바꿔 각 도형을 union 전에 복구. (spatialite `ST_MakeValid` 지원 여부 확인 필요 — 로컬 GDAL 3.7 및 클라우드 gdal-bin/spatialite 둘 다 테스트할 것)
2. 그래도 남는 오류 대비 **결과에 최종 makevalid 한 번 더**.
3. 대안/보강: dissolve도 안전변환처럼 **2단계**(① 원본 makevalid로 GPKG 생성 → ② 그 GPKG에서 `ST_Union GROUP BY`)로 만들면 spatialite `ST_MakeValid` 의존을 피할 수 있음. 이 방식이 더 견고할 수 있으니 우선 검토.
4. 관련 함수: `dissolve_one_layer`(app.py). 좌표변환 탭의 `convert_layer_safe`가 참고 패턴(2단계 makevalid).
