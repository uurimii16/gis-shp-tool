# 작업로그

SHP 좌표변환·병합·분할 도구 개발 기록. 상세 사양은 [README.md](README.md) 참고.

## 2026-07-03

기존 3탭(좌표변환·병합·분할) 기반 앱에 아래 기능을 추가·검증했습니다. 모든 항목은 로컬 QGIS 내장 GDAL 3.7로 실데이터 end-to-end 검증 완료.

### 신규 기능
- **변환 전후 feature count / extent 비교** — `ogrinfo` 출력을 파싱해 변환 전후 피처 수와 범위를 표로 비교.
- **도형 유효화 `-makevalid` 옵션** — 변환·병합·분할 공통. self-intersection 등 불량 도형 자동 보정.
- **병합(dissolve) 속성 집계 옵션** — 기준 컬럼 외 속성을 SUM/AVG/MAX/MIN/COUNT로 요약(컬럼별 지정).
- **작업 로그 다운로드** — 각 탭 처리 로그를 텍스트로 저장.
- **GDAL 미설치 안내 강화** — 설치/연결 방법을 사이드바에 단계별로 안내.
- **GPKG 내부 복수 레이어 선택** — `ogrinfo`로 GPKG 내부 레이어를 나열해 `파일명:레이어명` 단위로 개별 인식·선택.
- **면적 컬럼(area_m2, ㎡) 추가** — 변환·병합 탭. `ST_Area` 기반, 투영 후 결과에 후처리로 계산(미터 좌표계에서 정확). 경위도(4326)는 경고.
- **4. 코드 결합 탭(신규)** — SHP 속성의 MNUM 등에서 `substr(컬럼, 시작, 길이)`(기본 21,6)로 코드를 뽑아 용도지역 코드표 CSV를 LEFT JOIN. 원본 DBF를 직접 편집하지 않고 결합된 새 SHP/GPKG 생성.
  - 코드표 CSV 인코딩 후보별 깨짐 점수 표시 + 수동 선택(UTF-8-SIG/CP949/EUC-KR/UTF-8).
  - 대상 SHP 인코딩 확인, MNUM 컬럼/시작·길이 지정, 추출 코드·매칭률 미리보기.
- **안전 변환 모드(변환 탭, 기본 ON)** — 불안정 SHP를 다른 좌표계로 재투영할 때 폴리곤이 한두 개 사라지는 문제 방지.
  - ① 원본 좌표계 확정(`-a_srs`) + `makevalid`로 네이티브 좌표에서 도형 복구 → ② 재투영(`-t_srs`) + `makevalid` 2단계.
  - 입력 → 복구후 → 출력 단계별 피처 수 리포트 + 손실 발생 시 빨간 경고.

### 유지보수
- `use_container_width`(폐기 예정) → `width="stretch"` 전면 교체.
- 임시 작업 폴더를 시스템 Temp → 앱 폴더 내 `.runtime/`로 변경(권한 오류 회피).

### 클라우드 배포 후 수정
- **한글 손상(???) 버그 수정** — SQLite dialect로 SHP를 직접 쓰면 일부 GDAL 버전(클라우드 apt gdal-bin)에서 `-lco ENCODING`이 무시돼 한글이 ISO-8859-1로 손상됨. 공용 헬퍼 `sqlite_sql_to_output`으로 공간 SQL 결과를 UTF-8 GPKG로 만든 뒤 SHP로 재변환하도록 우회(면적/dissolve/코드결합 공통 적용). dissolve에 누락됐던 출력 인코딩도 지정.
- **면적 소수점 자리수 선택** — 변환·병합 탭에서 area_m2 소수점 0~6자리 지정.
- **결과물 빈손 방지 안전망** — SHP 저장이 실패하면 자동으로 GPKG로 대체 저장(`gpkg_to_final`). 변환·병합·분할 아님(분할 제외)·면적·코드결합 전 경로에 적용. 대체 발생 시 화면에 안내. 모든 처리 함수가 실제 저장 경로를 반환하도록 통일, 다운로드는 `download_for_path`가 실제 형식에 맞춰 제공.
